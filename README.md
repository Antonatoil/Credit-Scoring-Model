
# Credit Scoring Model

ML-проект для предсказания вероятности кредитного дефолта с использованием **CatBoost**, публикацией модели через **FastAPI**, логированием предсказаний в **PostgreSQL** и развёртыванием через **Docker Compose**.

## О проекте

Цель проекта — построить сервис кредитного скоринга, который принимает признаки клиента, оценивает вероятность дефолта и возвращает итоговый уровень риска.

Проект охватывает полный цикл работы с ML-моделью:

- загрузку и очистку данных;
- исследовательский анализ данных (EDA);
- обучение baseline-модели;
- обучение и настройку CatBoost;
- работу с несбалансированными классами;
- кросс-валидацию;
- сохранение модели и артефактов;
- развёртывание модели через FastAPI;
- логирование результатов предсказаний в PostgreSQL.

## Датасет

В проекте используется датасет **Default of Credit Card Clients**.

Он содержит информацию о клиентах кредитных карт, включая:

- кредитный лимит;
- пол;
- возраст;
- уровень образования;
- семейное положение;
- историю просрочек;
- суммы счетов;
- суммы предыдущих выплат.

Целевая переменная:

- `default_payment_next_month`
  - `0` — дефолта не было;
  - `1` — дефолт был.

## Архитектура решения

Проект построен по схеме **ML pipeline + service architecture**.

Основные части решения:

- **Data layer** — хранение сырых, промежуточных и обработанных данных;
- **Modeling layer** — подготовка данных, EDA, обучение и оценка моделей;
- **Artifacts layer** — сохранённые модели, метрики, thresholds и графики;
- **Serving layer** — FastAPI-сервис для инференса;
- **Persistence layer** — PostgreSQL для хранения логов предсказаний.

## Особенности реализации

### Работа с несбалансированными классами

Так как в задаче классы распределены неравномерно, в проекте использованы:

* `StratifiedKFold`;
* `class_weight="balanced"` для baseline-модели;
* `auto_class_weights="Balanced"` для CatBoost;
* подбор порога классификации по OOF-предсказаниям.

### Кросс-валидация

Для более устойчивой оценки использована **Stratified K-Fold cross-validation**.

### Подбор гиперпараметров

Для CatBoost реализован подбор гиперпараметров с выбором лучшей конфигурации по **PR-AUC**, так как эта метрика лучше отражает качество модели на несбалансированных данных.

### Публикация модели

После обучения модель сохранена и обёрнута в API на FastAPI. Сервис:

* принимает признаки клиента;
* возвращает вероятность дефолта;
* определяет итоговый класс риска;
* записывает результат в PostgreSQL.

## Результаты моделей

На тестовой выборке были получены следующие результаты:

| model    | threshold | accuracy | roc_auc  | pr_auc   | precision | recall   | f1       |
| -------- | --------- | -------- | -------- | -------- | --------- | -------- | -------- |
| catboost | 0.560524  | 0.796889 | 0.783944 | 0.561913 | 0.539589  | 0.554774 | 0.547076 |
| baseline | 0.588107  | 0.794444 | 0.712214 | 0.498528 | 0.542271  | 0.451256 | 0.492595 |

### Вывод по результатам

Модель **CatBoost** показала лучшие результаты по сравнению с baseline-моделью Logistic Regression.

Преимущества CatBoost:

* выше `ROC-AUC`;
* выше `PR-AUC`;
* выше `recall`;
* выше `F1-score`;
* меньшее количество пропущенных дефолтов.

Несмотря на небольшое увеличение числа ложноположительных срабатываний, CatBoost оказался более полезным для задачи кредитного скоринга, поскольку лучше выявляет рискованных клиентов.

## Логирование предсказаний

Каждое успешное предсказание сохраняется в PostgreSQL в таблицу `prediction_logs`.

Сохраняются:

* время запроса;
* вероятность дефолта;
* итоговый класс;
* уровень риска;
* исходный JSON-запрос.

Пример записи:

| id | created_at          | probability_default | predicted_class | risk_level |
| -- | ------------------- | ------------------- | --------------- | ---------- |
| 1  | 2026-03-12 14:53:40 | 0.8742579669066968  | 1               | high       |

## Как запустить проект

### 1. Подготовить датасет

Положить исходный датасет в папку:

```text
data/raw/
```

Рекомендуемое имя файла:

```text
default_of_credit_card_clients.xls
```

### 2. Обучить модель

Открыть и выполнить ноутбук:

```text
notebooks/01_credit_scoring_model.ipynb
```

После выполнения должны появиться файлы:

```text
models/catboost_credit_scoring.cbm
models/catboost_artifacts.json
```

### 3. Запустить проект через Docker Compose

```bash
docker compose up --build
```

### 4. Открыть Swagger UI

```text
http://localhost:8000/docs
```
